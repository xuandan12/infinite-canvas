# 快速开始

如果你只是想先把项目跑起来，优先部署或启动 `web/` 前端。

## Vercel 部署

在 Vercel 中导入仓库即可，根目录 `vercel.json` 会构建 `web/`。普通 OpenAI 兼容请求仍由浏览器直连；官方 Ark Seedance 2.0 的标准 API 与 Agent Plan 请求会由同源 Vercel Function 受限转发到固定的火山方舟地址，无需另建服务端。

## 本地启动

```bash
git clone git@github.com:basketikun/infinite-canvas.git
cd infinite-canvas
cd web
bun install
bun run dev
```

启动后访问：

```text
http://localhost:3000
```

## Docker 启动

如果你需要基于当前源码构建镜像：

```bash
docker build -t infinite-canvas .
docker run --rm -p 3000:3000 infinite-canvas
```

## 首次使用建议

- 先打开右上角配置弹窗，填入自己的 `Base URL`、`API Key` 和模型名。
- 如果需要提示词仓库内容，打开 `/prompts` 会由浏览器前端直连拉取并缓存到 IndexedDB。
- 如果需要跨设备同步画布、素材和生成记录，可在配置弹窗中填写 WebDAV。

## 说明

- 当前画布项目和“我的素材”主要保存在浏览器本地，WebDAV 同步需要用户自行配置。
- AI API Key 保存在浏览器本地。普通 OpenAI 兼容请求由前端直连；官方 Ark Seedance 请求只经当前站点的受限同源代理转发到火山方舟。
